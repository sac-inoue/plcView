# 002 PLCデータ収集アプリケーション要件定義書

## 1. 概要
本ドキュメントは、三菱電機製PLC（MELSECシリーズ）からTCP/IP（MCプロトコル）経由でデバイスデータを取得し、PCローカル環境へ保存するWindowsアプリケーションの仕様および要件を定義するものである。

> [!IMPORTANT]
> **【開発原則】NuGetパッケージ不使用方針**
> 本アプリケーションは外部ライブラリやNuGetパッケージ（Newtonsoft.Json、その他サードパーティ製通信ライブラリ等）を**一切使用せず**、Windows / .NET Framework 4.7.2に標準で同梱されているアセンブリ・ライブラリのみを用いて構築する。

---

## 2. システム前提条件・開発環境

| 項目 | 条件・要件 |
| :--- | :--- |
| **対象OS** | Windows 10 / 11 |
| **開発言語 / フレームワーク** | C# / .NET Framework 4.7.2 |
| **外部ライブラリ制限** | **NuGet使用不可** (.NET Framework 4.7.2 標準アセンブリのみで構築) |
| **対象PLC** | 三菱電機製 MELSEC Q / L / iQ-R / iQ-F シリーズ (内蔵Ethernet / 拡張Ethernetユニット) |
| **通信プロトコル** | MCプロトコル QnA互換3Eフレーム (TCP/IP バイナリ通信) |

### 2.1 NuGetを使わない（標準アセンブリのみの）実装方針
1. **ネットワーク通信 / MCプロトコル**:
   * `System.Net.Sockets.TcpClient` および `NetworkStream` を用い、バイナリ形式のMCプロトコルパケットの組み立て・解析を自作実装。
2. **JSONシリアライズ / デシリアライズ**:
   * `.NET Framework` 標準の `System.Web.Script.Serialization.JavaScriptSerializer` (参照: `System.Web.Extensions.dll`) または `System.Runtime.Serialization.Json.DataContractJsonSerializer` (参照: `System.Runtime.Serialization.dll`) を利用。
3. **GUIコンポーネント**:
   * Windows Forms (`System.Windows.Forms`) の標準コントロール（`TabControl`, `DataGridView`, `TrackBar`, `Timer` 等）を利用。

## 3. 機能要件

### 3.1 PLC通信設定要件
* **IPアドレス設定**: PLCのIPv4アドレス指定 (例: `192.168.3.39`)
* **ポート番号設定**: ポート番号指定 (例: `5007` / `1025`)
* **タイムアウト設定**: 通信受信タイムアウト・接続タイムアウト指定 (ミリ秒)

### 3.2 データ読出領域設定要件 (最大10ポイント)
プロジェクトごとに、読み出すデバイス領域を**最大10グループ(No.1〜No.10)**まで設定可能とする。

| 設定項目 | 内容 | 備考 |
| :--- | :--- | :--- |
| **有効/無効** | 該当ポイントの収集を行うかの有効フラグ | チェックボックス |
| **名称/メモ** | 領域識別用の任意テキスト | 例: 「ライン1 ステータス」 |
| **デバイス種別** | 対象デバイス指定 | 下表に示す三菱PLCの各種デバイスに対応 |
| **開始アドレス** | 読み出し開始のアドレス番号 | 10進数 / 16進数 (デバイスの進数仕様に準拠) |
| **サイズ (点数)** | 読み出すワード数 | 1〜960点 (1点=2バイト/16ビット) |

#### 3.2.1 対応デバイス一覧およびMCプロトコル仕様
主要な三菱PLCデバイス（ワードデバイスおよびビットデバイスのワード単位読出）に対応する。

| デバイス記号 | デバイス名 | 種別 | アドレス進数 | MCプロトコル コード (3Eバイナリ) |
| :--- | :--- | :--- | :--- | :--- |
| **D** | データレジスタ | ワード | 10進数 | `0xA8` |
| **W** | リンクレジスタ | ワード | 16進数 (Hex) | `0xB4` |
| **R** | ファイルレジスタ | ワード | 10進数 | `0xAF` |
| **ZR** | ファイルレジスタ (連番) | ワード | 16進数/10進数 | `0xB0` |
| **SD** | 特殊レジスタ | ワード | 10進数 | `0xA9` |
| **SW** | リンク特殊レジスタ | ワード | 16進数 (Hex) | `0xB5` |
| **TN** | タイマ (現在値) | ワード | 10進数 | `0xC2` |
| **CN** | カウンタ (現在値) | ワード | 10進数 | `0xC5` |
| **M** | 内部リレー | ビット | 10進数 | `0x90` |
| **X** | 入力 | ビット | 16進数/8進数 | `0x9C` |
| **Y** | 出力 | ビット | 16進数/8進数 | `0x9D` |
| **L** | ラッチリレー | ビット | 10進数 | `0x92` |
| **F** | アラーム | ビット | 10進数 | `0x93` |
| **B** | リンクリレー | ビット | 16進数 (Hex) | `0xA0` |
| **SB** | リンク特殊リレー | ビット | 16進数 (Hex) | `0xA1` |

* ※ビットデバイス（M, X, Y, B等）を16ビットまとめた「ワード単位読出し」に対応する。
* ※X, Y, B, W, SB, SW 等の16進数アドレス表記デバイス（例: `X100`, `B1A0`, `W100`）の入力・パケット変換をサポートする。

### 3.3 収集周期 (間隔) 設定
画面のドロップダウンリスト（ComboBox）等により、以下のプリセット間隔から選択可能とする。

| 設定値 | 間隔仕様 | 動作詳細 |
| :--- | :--- | :--- |
| **0 (最速)** | 内部 1 ms ウェイト | 応答を受信後、即座に次の要求を送信するが、PCのCPUフリーズを防止するため内部的に最低1msのウェイト（`Task.Delay(1)` 等）を必ず挟む |
| **1秒** | 1,000 ms | 1秒周期 |
| **2秒** | 2,000 ms | 2秒周期 |
| **5秒** | 5,000 ms | 5秒周期 |
| **10秒** | 10,000 ms | 10秒周期 |
| **30秒** | 30,000 ms | 30秒周期 |
| **1分** | 60,000 ms | 1分周期 |
| **5分** | 300,000 ms | 5分周期 |

* **タイマー / ループ制御**:
  * `0 (最速)` 選択時は、`Task.Run` 内の `while` ループで応答受領後に直ちに次のリクエストを発行する非同期連続処理とする。
  * 1秒以上の選択時は、指定時間タイマー (`System.Threading.Timer` 等) または `Task.Delay` により周期収集を行う。

### 3.4 プロジェクト設定保存・読み込み要件 (JSON形式)
* **設定のプロジェクト化**: 通信設定、データ領域設定(10ポイント分)、収集間隔、データ保存先パス等の全設定を「プロジェクト」として保存可能。
* **保存形式**: JSON形式 (`.json`)
* **操作**:
  * プロジェクトの新規作成
  * プロジェクトファイルの開く (`.json` 読み込み)
  * プロジェクトファイルの保存 (`.json` 上書き/別名保存)

### 3.5 データ保存要件
* **保存フォルダ指定**: 収集データの保存先フォルダをダイアログ等で自由に選択・指定可能。
* **保存形式**: 
  * バイナリ保存およびCSVテキスト保存に対応（設定で選択可能）。
  * **CSV出力の横持ちフォーマット**: Excel等でのグラフ解析を容易にするため、1回のサンプリングデータ（最大10ポイント分）を1行に横に並べて出力するクロス集計（横持ち）レイアウトとする。（例: `日時, Pt1_D0, Pt1_D1, Pt2_M0...`）
* **容量・ファイル自動分割要件およびファイル命名規則 (100MB上限 & 秒刻み命名)**:
  * **ファイル命名フォーマット**: 作成開始日時を秒まで含む `YYYYMMDD_HHmmss_プロジェクト名_連番.ext` 形式とする。（例: `20260720_100128_SampleProject_001.dat`）
  * **100MBサイズ分割**: 保存ファイルの容量が **100MB に達した時点で、新規作成時のタイムスタンプ(秒)と連番を付与した新しいファイルへ自動切替** する。（例: `20260720_100128_SampleProject_001.dat` -> `20260720_101530_SampleProject_002.dat`）
  * **目的**: 収集開始時刻や分割時刻を秒単位で即座に識別可能にし、ファイルの巨大化によるPCメモリ圧迫および履歴再生時の応答遅延を予防する。
* **差分保存オプションおよびデッドバンド機能 (選択可能)**: 
  * 前回値から変化があった場合のみログを書き出す機能。
  * **デッドバンド（しきい値）設定**: アナログセンサ等の微小ノイズによる無駄な保存を防ぐため、「前回値から ±〇〇 以上変化した場合のみ保存する」というしきい値を設定可能とする。

### 3.6 画面 (GUI) タブ構成および詳細要件
画面はタブコントロール (`TabControl`) により、各ビューを迅速に切り替え可能とする。

#### 1. 【設定・制御 タブ】
* 通信設定（IP、ポート、タイムアウト）
* 読出領域設定（最大10ポイントのグリッド編集）
* 収集周期・保存フォルダ指定
* プロジェクト操作（新規 / 開く / 保存）
* 収集開始 / 収集停止 ボタン、通信ステータス・ログ表示

#### 2. 【リアルタイム・メモリモニタ タブ】
* 収集実行中、PLCから取得した最新のメモリデータをグリッド（`DataGridView`）形式でリアルタイム表示。
* **表示単位・データ型変換切替 (表示モード)**:
  * **アドレス表示単位切替**:
    * **Word単位 (16bit)**: D0, D1, D2 ... としてワードごとに表示
    * **Byte単位 (8bit)**: 各ワードのLow Byte / High Byteを個別に分割表示 (例: D0-L, D0-H)
  * **データ表現切替**:
    * **数値表示**: 10進数 (符号あり/なし INT16/UINT16)、16進数 (Hex 0x0000)、2進数 (Binary 0b0000...)
    * **32ビット表示**: 連続する2ワードを DWORD / INT32 / 32bit浮動小数点(Float) として解釈表示
    * **ASCII文字列表示**: メモリ内のバイト列をASCII文字（英数字・テキストコード）にデコードして表示
* **視覚効果**: データ値が変化したセルを一時的にハイライト表示。

#### 3. 【履歴再生ビューア タブ (タイムスライダーモード)】
* 保存済みのデータファイル（CSV/バイナリ）を読み込んで過去のメモリ状態を再現表示。
* **タイムスライダー (TrackBar)**:
  * 録画・保存された時間軸（開始時刻〜終了時刻）に沿ってスライダーで直感的に刻刻と変わる時間を指定・移動。
  * スライダー移動に連動して、その時刻のメモリデータをリアルタイムに画面グリッド更新表示。
* **再生コントロール機能**:
  * 再生 / 一時停止 / コマ送り / コマ戻し ボタン
  * 再生速度切り替え (1倍速、2倍速、5倍速、10倍速)

### 3.7 デバッグログ保存機能 (デバッグモード)
* **デバッグモード切替**: UI上（設定タブ）に「デバッグモード有効（ログ保存）」のチェックボックスを配置。
* **ログ保存フォルダ**: アプリ実行ディレクトリ配下（またはプロジェクト指定領域）の `log` フォルダ（`./log/`）に自動保存。
* **ログ出力内容 (有効時)**:
  * 通信開始 / 停止、TCP接続・切断イベント
  * MCプロトコルの送信リクエストパケットおよび受信レスポンスパケット（バイナリHexダンプ表記）
  * 詳細な例外・エラーメッセージ、ターンアラウンドタイム(ms)
* **ログファイル命名**: `log/debug_YYYYMMDD.log` (1日1ファイル自動ローテーション)

## 4. 非機能要件

### 4.1 応答性・堅牢性
* 通信処理は別スレッド (Task / async) で行い、GUIスレッドをブロックしない。
* **タフな自動復旧（自己回復）ロジック**: LANケーブル抜去やPLCの電源再起動などの物理的な通信エラー・切断を検知した場合、内部のSocketを確実に破棄し、指定間隔（例: 5秒）で**無限に自動再接続（リトライ）を試み続ける**。現場での結線復旧後、一切のPC操作なしで自動的にデータ収集が再開される堅牢性を確保する。

### 4.2 メンテナンス性
* MCプロトコルパケット生成クラス、通信制御クラス、JSONファイル入出力クラス、GUI層を分離した設計とする。

### 4.3 外部ライブラリ非依存性（NuGet完全不使用）
* NuGetパッケージおよび外部DLLに依存しない設計とすることで、オフライン環境・セキュリティが厳しい工場内PCへの配備や、将来にわたるビルド環境の長期保守性を確保する。
