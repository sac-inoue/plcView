# plcView

三菱電機などのMCプロトコル（3Eフレーム）を使用して、PLCデータを収集、監視、および履歴再生するためのC# Windowsフォームアプリケーション（.NET Framework 4.7.2）です。

## 主な機能

- **MCプロトコル（3Eフレーム）対応**: TCP/IP経由で直接PLCと通信を行います。
- **柔軟なデータ収集設定**:
  - 最大10箇所のポイント（デバイス種別、開始アドレス、サイズ）を個別に有効・無効化し収集可能です。
  - 収集周期は0ms（最速・実測約1msウェイト）から5分まで選択可能です。
- **高信頼性・高パフォーマンスなログ保存**:
  - CSV形式（横持ちレイアウト）またはBinary（独自データストリーム）でのリアルタイム保存。
  - 1ファイル最大100MBでの自動ファイル分割機能。
  - 開きっぱなしストリームによるディスクI/O負荷低減。
- **データ再生（履歴再生機能）**:
  - 保存されたバイナリログファイルを読み込み、シークバーでタイムラインを自由に行き来できます。
  - 1倍速、2倍速、5倍速、10倍速の自動再生が可能。
- **描画パフォーマンス最適化**:
  - ダブルバッファリングによるグリッド描画のチラつき防止。
  - 行オブジェクトの再生成を抑え、セルの直接書き換えによるCPU負荷の大幅な削減。

## システム動作要件

- Windows 10 / 11
- .NET Framework 4.7.2
- MCプロトコル 3Eフレーム TCP通信が有効化されているPLC

## 使い方

1. **接続設定**: PLCのIPアドレス、ポート番号、タイムアウト値を入力します。
2. **ポイント設定**: 収集したいポイント（最大10点）のデバイス（D, W, M等）、開始アドレス、点数を設定します。
3. **データ収集の開始**: 「開始」ボタンをクリックすると、設定フォルダへログファイルが自動生成されつつ収集が始まります。
4. **履歴再生**: 「履歴再生」タブに移動し、作成された `.dat` バイナリファイルを読み込んで、再生コントロールからシークや倍速再生を行えます。

## ディレクトリ構成

- `plcView/Config/`: プロジェクト設定保存管理 (`ProjectSettings.cs`)
- `plcView/Plc/`: MCプロトコル通信クライアント (`McProtocolClient.cs`)、アドレス変換器 (`DeviceConverter.cs`)
- `plcView/Data/`: ログ出力制御 (`DataLogger.cs`)、バイナリインデクス解析 (`HistoryReader.cs`)
- `plcView/MainForm.cs`: UIイベント処理および収集ループスレッド制御
